Accessibility statement for the online COVID-19 (coronavirus) testing service

This accessibility statement covers the COVID-19 test websites available to citizens, including ordering rapid lateral flow tests for delivery and reporting rapid lateral flow test results.

We want as many people as possible to be able to use these websites. For example, this means you should be able to

  • change colours, contrast levels and fonts using browser functionality
  • zoom in up to 400% without the text disappearing off the screen
  • complete the service using only a keyboard
  • complete the service using speech recognition software
  • complete the service using a screen reader, for example JAWS, NVDA and VoiceOver

Weâ€™ve also made all text as simple as possible to understand.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible these websites are

We know some parts of the websites are not fully accessible.

Examples include:

  • the reCAPTCHA security check to prevent bots from using the service should be fully accessible most of the time, but may very occasionally cause issues for people using screen reader software
  • there are parts of the service that cannot be completed when JavaScript is disabled
  • some questions with radio button options may not be fully accessible to those using screen reader software

Feedback and contact information

If you have feedback or a complaint about the accessibility of this service, contact us online at https://enquiries.test-and-trace.nhs.uk/

You can also call 119. Check 119 opening times.

Technical information about the accessibility of these websites

We are committed to making these websites accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

These websites are partially compliant with the Web Content Accessibility Guidelines version 2.1 AA standard due to the non-compliances listed below.

Non-accessible content

The content listed below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

  • reCAPTCHA security check on the version of reCAPTCHA our service occasionally uses has an audio alternative, but this is not accessible to users. This fails WCAG 2.1 success criteria 2.1.1 (A), 2.1.3 (AAA), 2.5.6 (AAA), 4.1.1 (A)
  • There are parts of the service that cannot be completed when JavaScript is disabled while using Internet Explorer. This fails WCAG 2.1 success criterion 4.1.1 (A). The service was designed to operate in this way and there are no plans in place to update this
  • On the 'What is your home address?' page error messages are not announced by the screen reader. This fails WCAG 2.1 success criteria 2.4.3 (A), 3.3.1 (A)

What we're doing to improve accessibility

We are constantly iterating and improving our services to make them as accessible as possible. However, we might have missed something, so please let us know if you find any accessibility issues.

In early 2022 we commissioned a Web Content Accessibility Guidelines (WCAG) version 2.1 accessibility audit of the following website:

  • report a rapid lateral flow test result

The audit was carried out by Nomensa, who are accessibility specialists. They looked at a selection of pages most likely to have accessibility concerns. The audit found some accessibility issues, which we have now fixed.

We have also established accessibility processes, which we use as part of our ongoing work. These processes help us to design and develop accessible content whenever we make a change to any of our websites.

Preparation of this accessibility statement

This statement was prepared on 27 October 2020. It was last reviewed on 20 January 2023.

These websites were last externally audited for accessibility in February 2022. The test was carried out by Nomensa, The assessment identified some accessibility issues. We have fixed them all.

The accessibility of these services is reviewed on a regular basis. We will update this accessibility statement with any relevant changes.
